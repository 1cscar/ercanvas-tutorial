完整使用流程

Step 01

在 ERCanvas 設計
邏輯模型並匯出 SQL

在 ERCanvas 的邏輯模型模式中建立 ER 圖，設定好資料表與欄位後，點擊頂部工具列的 「匯出 MySQL SQL」按鈕。

善用「AI 自動正規化」功能讓模型自動優化，以及「外來鍵設定關聯主鍵」確保資料完整性。

💡 匯出時系統會自動帶入英文名稱，無需手動輸入欄位英文。
ERCanvas
Export
Step 02

自動產生 .sql
與中英翻譯對照檔

匯出完成後，系統會自動儲存兩份檔案到你的電腦：

  • *.sql — 可直接匯入 MySQL 的建表語法
  • *_zh_en_mapping.csv — 中英欄位名稱對照表
📂 檔案預設儲存於下載資料夾，可直接拖曳至 MySQL Workbench 使用。
Step 03

開啟 MySQL Workbench
選擇或建立連線

開啟 MySQL Workbench，在首頁的 MySQL Connections 中選擇目標資料庫連線（如 t1、t2、t3）。

🛡️ 如果擔心影響現有資料庫，建議點選「+」新建一個獨立連線，避免風險。
MySQL Workbench
Import
Step 04

匯入 .sql 檔案
載入建表語法

進入連線後，點擊左上角的「開啟 SQL 檔案」圖示（資料夾圖示），選擇剛剛匯出的 .sql 檔案。

編輯器中會顯示完整的 CREATE TABLE 語法，包含所有資料表結構。

⚠️ 確認頂部顯示的資料庫名稱（黃色框）是否正確。若名稱重複，請先修改 SQL 中的 DATABASE 名稱再繼續。
Step 05

點擊閃電圖示
執行並完成建表

確認 SQL 內容無誤後，點擊工具列上的閃電 ⚡ 圖示執行所有語法。

執行完成後，在左側 Schemas 面板即可看到所有建立好的資料表名稱，資料庫設計正式完成！

✅ 在左側 Tables 清單確認 driver、event、race 等表格都正確建立，即代表整個流程成功完成。
Execute
Edit
Step 06

直接在表格內
編輯欄位資料

若對自動翻譯的欄位名稱不滿意，可在 MySQL Workbench 的表格視圖中直接點擊編輯，無需重新設定所有數值資料。

✏️ 進入表格後一切皆可重新編輯，翻譯不準確時直接修改即可，不需重頭來過。
Step 07

外來鍵自動
連回對應主鍵

建立完成後，MySQL Workbench 會自動識別外來鍵關聯，並正確連結回對應的主鍵資料表，無需手動設定關聯。

🔗 在 ERCanvas 設定好的外來鍵關係，匯入 MySQL 後會完整保留，關聯結構自動對應。
Foreign Key
Step 08

點擊「Gemini 分析」
啟動 AI 正規化

完成基本流程後，可進一步點擊右上角的 「輸出 PDF 並交由 Gemini 分析」按鈕，讓 AI 重新審視並優化你的資料庫結構。

🤖 此步驟會將畫布內容轉為 PDF 並自動送交 Gemini，無需手動操作，全程自動化。
Gemini AI
Analysis
Step 09

確認分析流程
點擊「開始分析」

系統彈出說明視窗，顯示三個自動步驟：

  1. 把畫布輸出為 PDF 檔並下載到本機
  2. 用外接 Gemini 讀取 PDF 並輸出正規化建議
  3. 將 Gemini 建議轉成可建立的邏輯資料表
⚠️ 需事先在伺服器設定環境變數 GEMINI_API_KEY，可選設定 GEMINI_MODEL。
Step 10

查看正規化建議
建立實體圖

Gemini 分析完成後，視窗會顯示正規化後的資料表預覽，包含各表欄位建議（如：賽道、車手、車隊、賽事…）。

確認無誤後點擊「建立正規化實體圖」，自動產生優化後的 ER 圖。

✨ Gemini 會依資料領域自動判斷並建議合理的資料表數量與欄位，大幅減少人工設計時間。
