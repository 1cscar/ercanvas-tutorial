使用流程

五步驟完成
資料庫設計

透過 ERCanvas 強大的視覺化工具，搭配 MySQL Workbench，輕鬆完成從設計到部署的全流程。

Step 01

在 ERCanvas 設計
邏輯模型並匯出 SQL

在 ERCanvas 的邏輯模型模式中建立 ER 圖，設定好資料表與欄位後，點擊頂部工具列的 「匯出 MySQL SQL」按鈕。

善用「AI 自動正規化」功能讓模型自動優化，以及「外來鍵設定關聯主鍵」確保資料完整性。

💡 點擊「輸出 PDF 並交由 Gemini 分析」，系統會自動翻譯英文名稱，無需手動輸入英文欄位名。
ERCanvas
Export
Step 02

自動產生 .sql
與中英翻譯對照檔

匯出完成後，系統會自動儲存兩份檔案到你的電腦：

  • *.sql — 可直接匯入 MySQL 的建表語法
  • *_zh_en_mapping.csv — 中英欄位名稱對照表
📂 檔案預設儲存於下載資料夾，可直接拖曳至 MySQL Workbench 使用。
Step 03

開啟 MySQL Workbench
選擇或建立連線

開啟 MySQL Workbench，在首頁的 MySQL Connections 中選擇目標資料庫連線（如 t1、t2、t3）。

🛡️ 如果擔心影響現有資料庫，建議點選「+」新建一個獨立連線，避免風險。
MySQL Workbench
Import
Step 04

匯入 .sql 檔案
載入建表語法

進入連線後，點擊左上角的「開啟 SQL 檔案」圖示（資料夾圖示），選擇剛剛匯出的 .sql 檔案。

編輯器中會顯示完整的 CREATE TABLE 語法，包含所有資料表結構。

⚠️ 確認頂部顯示的資料庫名稱（黃色框）是否正確。若名稱重複，請先修改 SQL 中的 DATABASE 名稱再繼續。
Step 05

點擊閃電圖示
執行並完成建表

確認 SQL 內容無誤後，點擊工具列上的閃電 ⚡ 圖示執行所有語法。

執行完成後，在左側 Schemas 面板即可看到所有建立好的資料表名稱，資料庫設計正式完成！

✅ 在左側 Tables 清單確認 driver、event、race 等表格都正確建立，即代表整個流程成功完成。
